#  算法测试功能任务
    
    在 主菜单功能中，增加 算法测试 功能。
    

    算法测试:
            算法测试是一个弹窗界面(窗口)，主要功能为对图像的模型测试，
            通过
            选择模型， 根据目前 (配置文件目录)\model   中的  分类器， 目标检测 ， 分割模型 来改变部分页面
            选择目标文件夹，    用于获取测试图像的文件夹（递归扫描文件）
            输出文件夹。        （用于保存输出的文件夹）
            低置信度保存 阈值
            模式：  复制，移动
            多选：   进行分类保存 
                    保存标注文件
            进度条 执行

    逻辑：
        算法测试的模型为 后端使用到的 timm yolo 模型 api 通过返回模型名称来切换测试模型
        测试进行递归的扫描 目标文件夹的图片，使用模型进行识别。
        通过判断逻辑 置信度 < 阈值，重合检测框等，对问题图像进行保存
        目标检测/分割
            移动保在目标文件夹
                正常
                    空数据：
                        图像...
                    类别...
                    类别的保存逻辑为 "_".join(set(各类别名称))
                异常
                    低置信度
                        类别...
                        类别的保存逻辑为 "_".join(set(各类别名称))
                    重合检测框：
                        同类别重合
                            类别/图像
                        不同类别的重合
                            类别/图像
使用 websocket交流进度，进度条增加 检测速度 已完成/全部图像 预期时间

关于标注文件的保存：
    如果是 目标检测模型 标注文件使用 labelimg 的那种标签保留到文件
