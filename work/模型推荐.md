• 结合 gitmoudules/pytorch-image-models/results/results-imagenet.csv 的 Top‑1 精度，以及工业缺陷分类对特征表达和鲁棒性的需求，我给你挑了几档推荐，并用的是 timm
  里实际存在、在结果表里表现靠前的模型名。你可以直接用在 app.cli 的 --model 参数里。

  高精度（离线/算力充足场景）

  - c
      - ImageNet Top‑1 ≈ 88.2%，在卷积/ConvNeXt 系里属于非常高。
      - 有 In22k 预训练，适合小样本工业缺陷场景做微调，细节捕捉能力强。
      - 建议用于你数据较复杂、显卡 ≥ 24G、对实时性要求一般的任务。
  - convnext_large_mlp.clip_laion2b_soup_ft_in12k_in1k_384
      - Top‑1 ≈ 88.3%，在 results-imagenet.csv 里是卷积家族的顶级模型之一。
      - 基于 CLIP/LAION2B 大规模预训练，对“异常纹理”“细微划痕”这类分布偏离 ImageNet 的缺陷更友好。

  平衡型（精度 + 推理速度均衡，推荐首选）

  - convnextv2_base.fcmae_ft_in22k_in1k_384
      - Top‑1 ≈ 87.6%，相比 large 参数少不少，推理快很多。
      - 对工业缺陷细节仍然有不错的表征能力，也是我最推荐你先试的 baseline。
  - tf_efficientnetv2_m.in21k_ft_in1k
      - Top‑1 ≈ 86.0%，高效 CNN，吞吐量好，对边缘部署友好。
      - 适合你的数据量中等、显卡资源一般，但仍希望有较好精度的场景。
  - regnety_320.swag_ft_in1k
      - Top‑1 ≈ 86.8%，RegNetY 系列在推理 benchmark 里速度表现普遍不错。
      - 结构规则，适合量化/部署，对产线实时检测是一个很稳妥的选择。

  轻量但较强（实时/多路相机场景）

  - maxvit_small_tf_224.in1k
      - Top‑1 ≈ 84.5%，混合注意力 + 卷积结构，小模型中精度不错。
      - 当你需要高 QPS、多路相机并行、又要比传统 ResNet 更强一些时，可以选它。

  如何在你这个项目里用

  - 生成训练脚本：
    python -m app.cli --project-root . create-bat --plugin timm --model convnextv2_base.fcmae_ft_in22k_in1k_384 --data-dir datas\\your_vendor\\your_project
    --run-name convnextv2_base_defect

  如果你能告诉我显卡型号（比如 3060 / 3090 / A5000 等）、分辨率需求（如 224/320/384）和是否需要实时（单卡 QPS 大概多少），我可以从上面几个里帮你缩成 2–3 个最合适
  的“强基线 + 轻量备选”组合，并顺带给一套推荐的 batch size / 输入尺寸。
